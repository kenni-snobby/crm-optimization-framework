# CRM Reporting Framework

## Purpose

CRM reporting should help leaders make decisions.

Not create more dashboards.

The goal is visibility, accountability, and predictable revenue growth.

---

## Executive Dashboard

### Metrics

- Total Pipeline Value
- Pipeline Coverage Ratio
- Forecast Accuracy
- Revenue Won
- Win Rate

### Key Questions

- Are we on track to hit target?
- Do we have enough pipeline?
- Is forecast accuracy improving?

---

## Sales Manager Dashboard

### Metrics

- Opportunities Created
- Conversion Rates
- Pipeline Velocity
- Average Deal Size
- Deal Aging

### Key Questions

- Which reps need coaching?
- Where are deals getting stuck?
- Which stages need improvement?

---

## SDR Dashboard

### Metrics

- Calls Made
- Emails Sent
- Meetings Booked
- SQL Conversion Rate

### Key Questions

- Is activity producing outcomes?
- Which channels perform best?

---

## Pipeline Dashboard

Track:

- Stage Conversion Rates
- Opportunity Aging
- Lost Reasons
- Source Performance

---

## Revenue Dashboard

Track:

- Monthly Revenue
- Quarterly Revenue
- Revenue by Segment
- Revenue by Channel

---

## Reporting Principles

1. Measure outcomes before activity.
2. Every metric must support a decision.
3. Visibility drives accountability.
4. Simplicity beats complexity.
5. Review metrics consistently.

---

## Key Principle

Good reporting explains what happened.

Great reporting explains what to do next.
